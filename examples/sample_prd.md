# 智能配电监控系统（软硬件一体）PRD（简化版）

## 1. 产品目标
智能配电监控系统用于园区配电柜和边缘控制器的统一监控与远程控制，目标如下：
- 提供设备状态实时可视化（电压、电流、温度、开关状态）。
- 支持远程下发控制指令（合闸、分闸、参数下发）并获取回执。
- 在通信异常、设备重启、断电恢复等场景下保持状态一致性与可追溯性。
- 满足运维人员对告警、日志、故障诊断与快速恢复的需求。

## 2. 产品模块
- 网关通信模块：负责云端与边缘控制器的协议通信。
- 指令控制模块：负责指令下发、超时、重试、回执校验。
- 状态同步模块：负责设备状态上报、缓存、对账与纠偏。
- 告警与日志模块：负责告警规则、告警推送、操作审计、诊断日志。
- 设备升级模块：负责固件升级、失败回滚、版本校验。
- 权限模块：负责角色权限控制和关键操作二次确认。

## 3. 用户角色
- 系统管理员：管理设备、用户、策略，拥有全量权限。
- 运维工程师：查看状态、执行控制指令、处理告警。
- 观察者：只读查看，不允许执行控制操作。

## 4. 关键业务流程
1. 设备上线流程：
   设备上电 -> 网关注册 -> 平台鉴权 -> 状态上报 -> 进入在线态。
2. 控制指令流程：
   运维下发指令 -> 网关转发 -> 设备执行 -> 设备回执 -> 平台更新状态并记录日志。
3. 告警处理流程：
   设备触发告警 -> 平台推送 -> 运维确认 -> 告警闭环并保留审计记录。
4. 升级流程：
   平台下发升级任务 -> 设备下载并校验 -> 重启升级 -> 成功确认；失败则自动回滚并上报告警。

## 5. 非功能目标
- 指令回执超时阈值：默认 3 秒，可配置。
- 指令重试：最多 2 次，重试间隔 1 秒。
- 状态一致性：平台状态与设备状态最终一致，偏差修复不超过 10 秒。
- 关键操作（分闸、参数重置）必须记录审计日志。

## 6. 异常场景
- 通信超时：设备无回执或回执延迟。
- 指令重复执行：网络抖动导致回执丢失后重复下发。
- 状态同步失败：设备本地状态变化未成功上报，平台显示旧状态。
- 断电恢复：设备断电后重启，状态与任务队列需正确恢复。
- 升级失败：固件校验失败或升级中断，需触发回滚并告警。
